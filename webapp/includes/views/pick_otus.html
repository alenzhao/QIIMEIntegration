This script will assign each read in your library to an operational taxonomic unit (OTU). It can use a set of reference sequences describing the OTUs you expect to see, or it can create OTUs de novo

Here are scripts to execute before this one

  • Join paired ends - This script can only handle a single .fasta file. If you have two (one for each half of the pair), you must join them
  • De-multiplex libraries - This script is most effective if the headers in its input file include sample specific identifiers. De-multiplexing libraries ensures your input .fasta file will be correctly formatted

Here are the main decisions you should make before running this script

  1. Select your --input_seqs_filepath (typical .fasta, but headers should be formatted like de-multiplexing with QIIME formats them)
  2. Pick an --otu_picking_method
  3. Pick output options
    • uclust
    • uclust_ref
    • blast
    • Mothur
    • cd-hit
    • usearch
    • usearch_ref
    • usearch61
    • usearch61_ref
    • prefix/suffix
    • Trie

Here are the scripts to run after this one

  • Pick representative OTU sequence - For each OTU this script generates, it is efficient to select just one sequence that "represents" the whole OTU
  • Make OTU table - You don't technically need to pick representative sequences from each OTU before you create the OTU table, so you can jump straight to that step
