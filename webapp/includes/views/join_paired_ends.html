Some next generation sequencers (notably Illumina) produce paired end reads, generating two .fasta files whose corresponding reads are known to be a certain distance apart. This script combines paired ends into a single .fasta file, so that QIIME is better able to process them.

Here are scripts to execute before this one

  • None - You only need to use this script if you have paired end reads. It doesn't rely on any other scripts or QIIME specific files (e.g. a map file).

Here are the main decisions you should make before running this script

  1. Decide if you want/have an index read file (.fasta formatted file where the header corresponds to you input files, and the sequence is just the sample's barcode)
    • If you do, make sure the read order is the same in your input files, and that the header format is the same in both as well.
  2. Select your input files, --forward_reads_fp, --reverse_reads_fp, and (optionally) --index_reads_fp
  3. Decide on --min_overlap
  4. Pick a --pe_join_method, and specify corresponding options
    • fastq-join
    • SeqPrep
  5. Pick output options

Here are the scripts to run after this one

  • Validate mapping file - If you have not already completed this step, now would be a good time to do so.
  • De-multiplex libraries - Similar to this script, de-multiplexing is an important step (assuming your reads are multiplexed) before you can begin the rest of your analysis.
  • Pick OTUs - If you do not have to demultiplex your data, you can skip straight to OTU picking
