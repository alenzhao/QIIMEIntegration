This is the final step in QIIME's OTU analysis. It creates a table that stores read counts per OTU for each sample. The file it creates can be exported to various other software packages for further analysis

Here are scripts to execute before this one

  • Pick OTUs - This script uses an OTU map file (each line is an OTU, each column is a read), which is generated by pick OTUs
  • Assign taxonomy - If you want to include real-world taxa in your OTU table, you must first assign your OTUs

Here are the main decisions you should make before running this script

  1. Select your input --otu_map_file (the result of Pick OTUs)
  2. Specify a taxonomy file, if you have created one
  3. Decide if you want to exclude any OTUs. If you do, create a plain text file with one OTU per line, and upload it for use in the paramter --exclude_otus_fp
  4. Pick output options

Here are the scripts to run after this one

  • Manipulate OTU table - Once you have created your OTU table, you can run various commands on it, for example, file format conversion
  • Sequence alignment - Now you may want to go on to perform phylgeny analysis
