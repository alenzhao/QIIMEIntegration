It is highly economical to sequence multiple samples on a single NGS run. This script splits reads from a single fasta file (a single run) into their respective samples. It also can perform quality filtering at the same time

Here are scripts to execute before this one

  • Validate mapping file - This script relies on a mapping file to accurately tell which samples correspond to which sequence barcodes. You may want to check the format of your file before running this script
  • Join paired ends - This script only works on a single fasta file. QIIME provides you a script to join paired end reads into a single file

Here are the main decisions you should make before running this script

  1. Select input options
  2. Pick options for de-multiplexing
  3. Pick options for filtering
  4. If you have a quality file, pick additional options for quality filtering
  5. Pick output options

Here are the scripts to run after this one

  • Pick OTUs - Once your multiplexed reads have been assigned to samples, you are ready to select OTUs

Here are some of the common pitfalls using this script

  • As far as barcode types, golay_12 does not refer to all 12-length barcodes
