Here are scripts to execute before this one

  • Join paired ends - This script can manage either paired end, single end, or combined .fastq files. The rest of QIIME cannot handle non-joined paired-ends. You can run Join paired ends either before, or after this script. If you run it before, use --input_type = paired_end_stitched
  • Validate mapping file - This script optionally takes a mapping file as input. You should validate it before you use it here

Here are the main decisions you should make before running this script

  1. Pick your --input_type
  2. Specify your one or two input .fastq files (depends on --input_type)
  3. Decide on barcode options
    • Length of one or both barcodes
    • Do you want to attempt reordering based on a map file?
    If working with two .fastq files, do you want to --disable_header_match
  4. Pick output options
    • Do you want to reverse complement barcodes before printing them?

Here are the scripts to run after this one

  • De-multiplex libraries (fastq) - This script generates a new .fastq file that has just barcodes as its sequences

Here are some of the common pitfalls using this script

  • Be sure to specify the correct barcode length. The default length is 6 bases, which is frequently incorrect
  • You may have to reverse complement the barcodes. We recommend using a --mapping_fp to perform the orientation for you.
    • If you have paired ends, this only works if your map file has ReversePrimers
